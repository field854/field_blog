# 2026 年度计划

## 一、2026 年技术方向

> **工程 + 编程的“可交付技术路线**

### 为什么选这条
- 能比赛（电赛）
- 能做项目
- 能远程
- 不依赖名校、不卷学历
- 尽快变现或实习

---

## 二、2026 年核心技术栈

### 【一级核心】必须掌握

#### 1. C / C++（工程向，不是算法向）
- 用途：嵌入式、系统、工程逻辑
- 学到什么程度：
  - 指针 / 结构体 / 内存概念清楚
  - 能写 **完整工程代码**，不是刷题
- 目标状态：
  - 看到需求 → 能写程序实现

---

#### 2. Linux（工程环境）
- 用途：开发、部署、远程工作必备
- 学到什么程度：
  - 常用命令熟练（cd / ls / grep / vim）
  - 会用 shell 跑程序、查日志
- 目标状态：
  - 不害怕终端
  - 能在 Linux 下完成完整开发流程

---

#### 3. Git（版本控制）
- 用途：项目、比赛、远程协作
- 学到什么程度：
  - clone / add / commit / push / pull
  - 会用 GitHub 作为作品展示
- 目标状态：
  - 每个项目都有仓库
  - 能给别人看你做过什么

---

### 【二级核心】电赛 & 工程能力（主战场）

#### 4. 嵌入式 / 单片机（电赛核心）
- 推荐路线（二选一即可）：
  - STM32
  - Arduino / ESP32
- 学到什么程度：
  - 会用 GPIO / 串口 / 定时器
  - 能接传感器、控制执行器
- 目标状态：
  - 能独立做出“会动、会反馈”的系统

---

#### 5. 基本电子与硬件理解
- 不追求理论极深
- 重点是：
  - 能看懂电路图
  - 知道模块怎么接、为什么能用
- 目标状态：
  - 不怕硬件
  - 出问题知道从哪查

---

### 【三级辅助】为“能工作”服务

#### 6. Python（工具与自动化）
- 用途：
  - 数据处理
  - 自动化
  - 写小工具
- 学到什么程度：
  - 能写脚本解决实际问题
- 目标状态：
  - 把重复工作自动化

---

#### 7. 技术英语（功能型）
- 不学“英语”
- 只学：
  - 看 datasheet
  - 查英文文档
  - 写简单技术说明
- 目标状态：
  - 遇到英文资料不逃避

---

### 【明确降级】2026 年不作为主线的东西

- ❌ 算法竞赛（ICPC） → **保温即可**
- ❌ 多语言乱学
- ❌ 高深理论
- ❌ 完美主义

---

## 三、技术栈 × 时间轴（2026 年怎么学）

---

### 【2026.01 – 03】技术启动期
**重点**
- C / C++
- Linux 基础
- Git 入门

**成果**
- 一个能跑的小程序
- 一个 GitHub 仓库

---

### 【2026.04 – 06】工程成型期
**重点**
- 单片机 / 嵌入式
- 硬件 + 软件联动

**成果**
- 一个完整小系统
- 能演示、能讲清楚

---

### 【2026.07 – 08】现实验证期（关键）
**重点**
- 项目完整度
- 稳定性
- 可复现

**成果（至少一个）**
- 电赛完整项目
- 或工程项目交付
- 或实习 / 项目组经历

---

### 【2026.09 – 12】稳态提升期
**重点**
- 项目迭代
- 技术体系整理
- 为工作做准备

**成果**
- 作品集
- 明确技术定位

---

## 四、每天干什么

### 每天只做这 3 类之一即可

- 写一段工程代码  
- 调试一个模块  
- 查一份技术资料并记 5 行笔记  

> **只要和上面技术栈有关，就算赢**

---

## 五、2026 年底你应该变成什么样

- 不再说“我学过”
- 而是：
  - **“我做过这个”**
  - **“这个项目是我独立完成的”**

这才是你能：
- 尽快工作
- 租房
- 养狗
- 宅着活下去  
的真正底气。
